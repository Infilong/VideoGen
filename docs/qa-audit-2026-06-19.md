# HighlightAI Functional Audit - 2026-06-19

## Result

Status: PASS WITH LIMITATIONS

The local workflow is functional after fixes to refinement, playback, score calibration,
persistence, large-library rendering, AI setup errors, and modal accessibility.

## Fixed During Audit

- Refine now rebuilds segments, increments the draft version, clears stale exports, and
  starts a new background render.
- Older renders cannot overwrite a newer refined draft.
- Refine requires an explicit instruction instead of silently applying "More cinematic."
- Electron hardware acceleration is enabled for AV1/H.264 gameplay playback.
- Source playback uses byte-range streaming.
- Local highlight scores are calibrated across the project instead of saturating at 100.
- Score badges identify `Local` signals separately from actual `Vision AI` results.
- Concurrent project/job writes use serialized atomic JSON replacement.
- Large libraries initially render 40 cards and progressively load more.
- Missing Ollama models return an actionable install command.
- Settings, Assets, preview, and notification close controls have accessible names.
- Escape closes open modals.
- The clip library supports grid, compact list, and detailed views with persistent choice.
- Filtered clips can be selected in bulk up to the 20-clip generation limit or cleared.
- Soundtrack selection is a visible required step, with local and public music in one panel.
- Rendering always receives the explicitly selected audio asset.
- Dark and light themes are available and persist locally.
- React UI tests now clean up after every case to prevent leaked state between tests.

## Rating Semantics

- `Local N`: relative highlight score from motion, scene changes, audio peaks, and
  technical quality. It is not a semantic AI opinion.
- `AI N`: score produced only after a configured multimodal model reviews sampled frames.

The Battlefield project currently has 138 local-signal ratings ranging from 38 to 95,
with 58 distinct values. It has no completed vision-AI ratings.

## Limitations

- Vision preprocessing and the multi-reviewer AI workflow require a running compatible
  vision model. They were not executed because no configured model was available.
- Date-only filenames cannot produce reliable object tags such as tank, helicopter, or
  explosion from local audio/motion signals. Those tags require vision preprocessing.
- Public audio search was tested; downloading every third-party result was not performed.
- No packaged build was created.
